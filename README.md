# Resume Studio

一个本地优先的网页端简历工作台，用于导入简历、在线编辑排版、岗位优化、头像处理、面试 PPT 生成，以及辅助招聘网站投递填表。

> GitHub 会自动识别并展示 `README.md`。本项目说明文档只保留这一份，便于公开发布和维护。

## 功能亮点

- 本地运行：静态网页应用，无需数据库和后端服务。
- 多格式导入：支持 TXT、MD、JSON、CSV、PDF、DOCX、RTF、HTML 等常见简历文件。
- 直接编辑：既可以在左侧表单编辑，也可以直接点击右侧简历预览文字进行修改。
- 成熟度检查：内置基础信息、摘要、技能、经历、项目、教育证书等结构检查。
- 本地整理：支持摘要整理、技能规范、经历格式整理和一页简历压缩。
- 多版式排版：提供专业双栏、原简历风格、Markdown 单栏、紧凑单页、作品展示、ATS 纯文本。
- 样例库：内置前端高级、嵌入式 Linux、AI 产品工程、B 端产品经理、数据分析师、校招后端等样例。
- AI 优化：支持 OpenAI Responses API、DeepSeek、通义千问兼容、智谱 GLM、Kimi/Moonshot 和自定义兼容接口。
- 智能投递：可根据招聘页字段标题进行本地/AI 匹配，处理项目经历、获奖荣誉、证书等同义字段。
- PPT 页面：可基于简历生成 2-8 页的面试介绍或作品集，并在线编辑。
- 本地保存：支持保存为 `.resume.json` 工作文件，下次打开后继续编辑。

## 快速开始

### Windows 推荐方式

1. 下载或克隆本项目。
2. 进入 `resume-studio` 文件夹。
3. 双击 `open-resume-studio.bat`。
4. 浏览器会打开：

```text
http://127.0.0.1:8765/index.html
```

保持命令窗口不要关闭；关闭后本地网页服务会停止。

### 直接打开

也可以直接双击 `index.html` 使用。MD、TXT、JSON、RTF、DOCX 和带文本层的 PDF 均可导入；复杂 PDF 建议使用本地网页服务方式打开。

### macOS / Linux

在项目目录中运行：

```bash
python3 -m http.server 8765
```

然后访问：

```text
http://127.0.0.1:8765/index.html
```

公开发布时建议只保留以下文件：

resume-studio/

## 使用流程

1. 打开页面。
2. 在顶部样例下拉框选择一个版本，点击 `载入样例` 熟悉界面。
3. 点击 `导入`，选择现有简历文件，或直接在左侧编辑内容。
4. 在 `简历` 面板检查成熟度，使用 `整理摘要`、`规范技能`、`整理经历` 或 `压缩一页`。
5. 在 `优化` 面板选择目标方向，粘贴岗位 JD，点击 `本地优化` 或配置 API 后点击 `AI优化`。
6. 在 `版式` 面板选择模板、主题色、字号、行距和显示模块。
7. 点击 `另存为` 保存 `.resume.json` 工作文件；之后可用 `打开` 继续编辑。
8. 点击 `打印/PDF`，使用浏览器打印功能导出 PDF。

## 内置样例

- 前端高级：Markdown 单栏，适合技术型候选人展示完整项目和工程化能力。
- 嵌入式 Linux：专业双栏，突出 BSP、驱动、通信协议和量产调试。
- AI 产品工程：专业双栏，适合 AI 工具、前端工程和产品落地复合背景。
- B 端产品经理：作品展示风格，强调业务流程、产品方案和结果指标。
- 数据分析师：紧凑单页，突出指标体系、SQL/Python、实验分析和看板能力。
- 校招后端：ATS 纯文本，适合应届生和网申系统快速识别。

## 文件导入

- `.md`、`.txt`、`.csv`、`.html`：直接读取文本。
- `.json`、`.resume.json`：读取 Resume Studio 保存的结构化工作文件。
- `.pdf`：优先使用内置 pdf.js 抽取文本和坐标；带文本层的双栏/侧栏 PDF 会尝试识别原文件结构。
- `.docx`：使用内置 ZIP/XML 抽取器读取正文。
- `.rtf`：进行基础去格式处理后抽取文本。
- `.doc`：会尝试旧版 Word/OLE 文档文本抽取；失败时建议先另存为 `.docx` 或 PDF。

扫描版或图片型 PDF 没有可复制文本层，浏览器内无法稳定 OCR，建议先用 WPS、Adobe Acrobat 或 OCR 工具识别成可复制文字后再导入。

## AI API 配置

在 `优化` 面板填写：

- 服务商：OpenAI、DeepSeek、通义千问兼容、智谱 GLM、Kimi/Moonshot 或自定义。
- 接口类型：OpenAI Responses 或 Chat Completions 兼容。
- 接口地址：填写你的模型服务地址。
- 模型：填写你的账号可用模型。
- API Key：仅在当前页面运行时使用，默认不持久化保存。

生产环境建议增加后端代理：

1. 前端请求自己的后端接口，例如 `/api/resume-optimize`。
2. 后端从环境变量读取模型密钥，例如 `MODEL_API_KEY`。
3. 后端调用 OpenAI、DeepSeek 或其他兼容模型服务。
4. 前端不保存、不展示真实 API Key。

## 投递助手

投递功能不会自动提交申请，只辅助生成和填入内容。

1. 在 `投递` 面板补充姓名拆分、LinkedIn、期望岗位和求职信摘要。
2. 如果投递页有“项目经验、代表项目、获奖情况、证书资质”等非标准字段，把页面小标题粘贴到 `投递页字段 / 小标题`。
3. 点击 `本地匹配`，或配置 API 后点击 `AI匹配`。
4. 点击 `重新生成脚本` 或 `准备投递`。
5. 复制脚本到招聘页面 Console 运行，或使用一键填表书签。
6. 人工核对所有字段、附件、隐私授权和验证码后，再手动提交。

AI 匹配会把当前简历和投递页字段标题发送到你配置的模型接口。

### 字段采集脚本怎么用

当招聘网站字段标题和简历字段不一致时，例如“代表项目”“项目经验”“获奖情况”“证书资质”，建议先采集页面字段：

1. 在 Resume Studio 的 `投递` 面板点击 `复制字段采集脚本`。
2. 打开招聘网站，进入真正填写简历信息的表单页。
3. 按 `F12` 或 `Ctrl + Shift + J` 打开开发者工具。
4. 切换到 `Console` 控制台。
5. 粘贴字段采集脚本并回车运行。
6. 如果浏览器提示不允许粘贴，先确认光标在 `Console` 最底部的输入行里，然后输入：

```text
allow pasting
```

然后回车，再重新粘贴脚本运行。

如果输入 `allow pasting` 后依旧不能粘贴，通常是浏览器、公司策略或页面安全限制导致。可以改用下面任意一种方法：

- 方法一：在开发者工具里打开 `Sources` / `源代码`，找到 `Snippets` / `代码段`，新建一个代码段，把字段采集脚本粘贴进去，按 `Ctrl + Enter` 运行。
- 方法二：不要运行脚本，直接把投递页面看到的小标题复制到 Resume Studio 的 `投递页字段 / 小标题`，每行一个，例如 `项目经历`、`获奖情况`、`证书资质`、`个人优势`。
- 方法三：如果脚本运行后没有自动复制，它会在 Console 打印 `Resume Studio 字段 JSON`，并弹出一个可手动复制的窗口，把里面的 JSON 复制回 Resume Studio 即可。

7. 脚本会扫描当前页面可见输入框、label、placeholder 和附近小标题，并复制一段 JSON。
8. 回到 Resume Studio，把 JSON 粘贴到 `投递页字段 / 小标题`。
9. 点击 `本地匹配` 或 `AI匹配`。
10. 检查 `智能匹配结果`，确认项目经历、获奖情况、证书资质等字段是否匹配正确。
11. 点击 `重新生成脚本` 或 `准备投递`，再执行自动填表脚本。

字段采集脚本只读取当前页面字段结构，不会提交申请，也不会上传附件。多步骤投递表单建议每进入一个新步骤后重新采集一次。

## 数据与隐私

- 简历内容默认只存在浏览器内存中。
- `另存为` 会生成你自己选择位置的 `.resume.json` 工作文件。
- 已通过 `打开` 或 `另存为` 绑定本地文件时，`保存` 会写回当前文件。
- 未绑定本地文件时，`保存` 会写入当前浏览器的 `localStorage`。
- 导入 `.resume.json` 时，如果文件中包含 API Key，页面会自动清空。
- 自动填表脚本不会点击提交按钮。

公开发布前请不要把以下内容放进仓库：

- 真实 API Key、账号密码、Cookie 或 Token。
- 真实简历、身份证号、手机号、投递记录。
- `.resume.json` 私人工作文件。
- PDF/Word 简历原件、截图、浏览器缓存。
